1. Setup Inicial (/setup)

Rode uma única vez para configurar tudo:

/setup
O que acontece:
  1. Conecta uma pasta de trabalho (onde ficam config, leads, sites)
  2. Coleta sua assinatura (nome, apresentação, WhatsApp)
  3. Define nichos padrão e cidade
  4. Configura aapanel: abre o dashboard → aba Configurações → preenche URL, Token API, Domínio base, Usuário SSH, Senha SSH
  5. Configura Playwright (headless, stealth, rate limit)
  6. Cria banco SQLite (prospector.db) e dashboard
  7. Copia scripts do publicador automático para a pasta
  8. Instala o publicador como serviço (systemd no Linux/Mac, Task Scheduler no Windows)
⚠️ Importante: A senha SSH e o Token API do aapanel NUNCA são digitados no chat. Você preenche na aba Configurações do dashboard (interface web local) → clica "Salvar" → vão direto pro prospector-config.json no seu computador.
✅ Pronto quando: aparecer "Dashboard atualizado: X leads" e o link http://localhost:8765 abrir o painel.

2. Prospecção (/prospectar)

/prospectar [nicho] [cidade] /prospectar nutricionistas "São Paulo" /prospectar
O que faz:
  1. Lê prospector-config.json (nichos, cidade, nota mínima, avaliações mínimas, meta de leads)
  2. Inicia Playwright headless no servidor (Chromium + stealth)
  3. Busca no Google Maps: [nicho] em [cidade]
  4. Scroll infinito + extrai cards (nome, nota, avaliações, site, telefone, endereço)
  5. Filtros eliminatórios:
    • Nota ≥ 4.7 E avaliações ≥ 40 (potencial financeiro)
    • Tem site ativo (não diretório terceiro, não fora do ar)
    • Site ruim (≥ 2 problemas: layout antigo, sem CTA, domínio grátis, não responsivo, conteúdo bagunçado, sem prova social)
    • Tem e-mail público (site, Google search fallback)
  6. Para cada lead qualificado: extrai WhatsApp (link wa.me no site OU celular do Maps com 9º dígito)
  7. Salva no Google Sheets (via Drive API) + leads.md local + Dashboard SQLite
📋 Saída obrigatória: Link da planilha Google + tabela no chat + "Dashboard atualizado: N leads" + sugestão /redesenhar para os 5+ melhores.

3. Redesign Premium (/redesenhar)

/redesenhar [cliente ou "todos"] /redesenhar "Dra. Maria Silva" /redesenhar todos
Gera 3 arquivos em sites/[slug]/:
  1. index.html — Página final: Hero com CTA WhatsApp, Sobre, Serviços (cards), Depoimentos (carrossel), Equipe, FAQ (schema), Contato/Mapa, Footer. Mobile-first, SEO on-page, Core Web Vitals otimizados.
  2. editor.html — Editor visual: clique para editar textos, trocar imagens, ajustar cores/tipografia via variáveis CSS, drag-and-drop de seções, preview mobile/desktop, botão "Exportar HTML final".
  3. comparador.html — Slider antes/depois (desktop + mobile) com marcadores visuais das melhorias.
Atualiza status: novo → redesenhado no leads.md + dashboard.

4. Publicar no aapanel (/publicar)

/publicar [cliente ou "todos"] /publicar "Dra. Maria Silva" /publicar todos
Fluxo (tenta API direta primeiro):
  1. Gera proposta.html (capa com antes/depois + assinatura) em sites/[slug]/
  2. Método 1 - API aapanel: Cria site → SSL Let's Encrypt → Upload via SFTP/rsync → Verifica HTTPS
  3. Método 2 - Publicador local (fallback): Escreve fila-publicacao.txt → serviço systemd/Task Scheduler pega a cada 60s → rsync → renomeia fila → log em publicador-log.txt
  4. Verificação HTTPS BLOQUEANTE: curl -sSf https://dominio/ deve retornar 200 com cadeado válido. http:// NUNCA vai pro cliente.
Estrutura no servidor: 
Subdomínio (padrão):    https://cliente-slug.seudominio.com/
Subpasta (opcional):    https://seudominio.com/clientes/cliente-slug/
Saída: URLs testadas (página + capa) + sugestão /proposta.

5. Enviar Proposta (/proposta)

/proposta [cliente ou "todos"]
E-mail criado como rascunho no Gmail (padrão) ou enviado direto:
  • Assunto: Pergunta pessoal + nome do negócio (≤ 60 chars, sem emoji, sem CAIXA ALTA)
  • Corpo: 120-180 palavras. Elogio específico (nota/avaliação real) → 1-2 pontos objetivos do site atual → "preparei nova versão, já no ar" → ÚNICO link = https://.../proposta.html (âncora HTML com URL limpa visível) → CTA: "abre no celular e me diz o que achou"
  • Assinatura: Nome + apresentação + WhatsApp do config
🚫 Checklist anti-spam (BLOQUEANTE):
  • 1 link só (a capa). Máx 2 se incluir site antigo.
  • Zero encurtador (bit.ly = spam na certa).
  • Link como âncora HTML: https://seudominio.com/... (texto visível = URL limpa do config).
  • Domínio apresentável (não subdomínio técnico cheio de números). Se for feio → PARE e ative domínio próprio no aapanel.
  • Sem palavras-gatilho: grátis, promoção, imperdível, oferta, desconto, clique aqui, 100%, garantido, urgente.
  • Sem CAIXA ALTA, "!!", emoji no assunto.
  • HTML minimalista (só parágrafos + âncora). Zero cores, botões, imagens, anexos.
  • Assunto ≤ 60 chars, pergunta ou frase pessoal com nome do negócio.
  • Primeira linha 100% personalizada (nome + fato real das avaliações).
  • Remetente = Gmail pessoal ativa do usuário (SPF/DKIM do Google).
Follow-up automático: /followup após 3+ dias úteis sem resposta (1 único por lead).

6. Dashboard de Leads

Duas formas de acessar:
  1. Modo Servidor (completo): ./iniciar-dashboard.sh (Linux/Mac) ou iniciar-dashboard.bat (Windows) → abre http://localhost:8765 com Kanban drag-drop, edição inline, exclusão, busca, paginação, funil, follow-ups, financeiro, contratos — tudo salvando no SQLite.
  2. Modo Arquivo (fallback): Duplo clique no dashboard.html → abre no navegador, só leitura + edições presas ao localStorage.
Visões (abas no topo):
  • Kanban: Colunas por status (Novo → Redesenhado → Publicado → Proposta → Respondeu → Fechado → Descartado). Arrasta cards para mudar status.
  • Contratos: Tabela com status (Pendente/Enviado/Assinado), link do documento, pago sim/não.
  • Financeiro: Recebido, A Receber, MRR manutenções, Projeção 12 meses.
Atualização automática: Todo comando que muda lead (/prospectar, /redesenhar, /publicar, /proposta, /contrato) faz upsert no SQLite + regenera dashboard.html com snapshot JSON embutido.

7. Editor Visual (/editor)

/editor "Nome do Cliente"
Abre sites/[slug]/editor.html no navegador (Claude in Chrome). Edita textos clicando, troca imagens, ajusta cores/tipografia via painel lateral, reordena seções, preview mobile/desktop. Botão "Salvar HTML final" baixa o index.html atualizado. Se já publicado, rode /publicar depois.

8. Follow-up (/followup)

/followup [cliente ou "todos"]
Encontra leads com status proposta há ≥ 3 dias úteis sem respondeu/fechado. Cria rascunho curto: "Oi [Nome], conseguiu ver a página? Qualquer dúvida tô à disposição." Mesmo thread se possível. Registra data do follow-up no lead.

9. Contrato (/contrato)

/contrato "Nome do Cliente"
Gera contrato Markdown/PDF com: partes, objeto (site + deploy + SSL + página proposta + treinamento), valor/parcelas/manutenção, prazos, obrigações, propriedade intelectual (código = seu, conteúdo = do cliente), rescisão, foro. Salva em contratos/[slug]-contrato.md. Atualiza dashboard: contratoStatus=enviado, contratoEm=hoje. Cliente assina → assinado → pagamento → pago=1.

12. Problemas Comuns

Playwright: CAPTCHA/login do Google Maps
Na primeira execução, pode pedir login. Resolva manualmente no Chrome que abrir → estado salvo em playwright-state.json → próximas rodadas passam direto.
aapanel: SSL falha / Let's Encrypt erro
No painel aapanel → SSL → marque o domínio → "Run AutoSSL". Aguarde alguns minutos. Se persistir: verifique se domínio aponta pro IP do servidor (A record) e porta 80/443 abertas no firewall.
Deploy: rsync/SSH falha
Teste manual: ssh usuario@seu-servidor. Se pedir senha, configure chave SSH (ssh-copy-id) ou use senha no config. Verifique se usuário tem permissão de escrita em /www/wwwroot/.
E-mail: proposta não abre / link estranho
Verifique no rascunho do Gmail: o texto visível do link deve ser https://seudominio.com/.... Se aparecer google.com/url?q=... → o link não está como âncora HTML. Corrija no template.
